# TCP 多用户聊天系统 (Python版)

基于 TCP 协议的多用户网络聊天系统，支持用户注册/登录、群聊、私聊、聊天记录保存等功能。

## 功能特性

### 用户管理
- 用户注册（需检查用户名唯一性）
- 用户登录（验证用户名和密码）
- 在线用户实时列表

### 聊天功能
- **群聊**：发送消息给所有在线用户
- **私聊**：选择在线用户进行一对一聊天
- 用户上下线通知
- 聊天记录保存到数据库

### 图形界面
- 使用 tkinter 构建的 GUI
- 登录/注册窗口
- 聊天主窗口（左侧用户列表 + 右侧聊天区域）
- 消息分类显示（群聊/私聊/系统消息）

## 文件结构

```
.
├── database.py      # 数据库管理模块
├── server.py        # 服务器端程序
```

## 启动说明

### 1. 启动服务器

打开终端，运行服务器程序：

```bash
python server.py
```

服务器启动后显示：
```
==================================================
    TCP 多用户聊天服务器
==================================================

服务器配置: 0.0.0.0:8888
聊天服务器启动成功!
监听地址: 0.0.0.0:8888
按 Ctrl+C 停止服务器
==================================================
```

### 2. 启动客户端

另开终端，运行客户端程序：

```bash
python client.py
```

客户端会弹出登录窗口，填写服务器地址、端口、用户名和密码即可登录。

### 3. 测试多用户聊天

1. 启动服务器
2. 启动多个客户端（每个客户端使用不同的用户名）
3. 在任一客户端输入消息，其他客户端都能收到（群聊）
4. 点击左侧用户列表中的用户名，可以切换到私聊模式

## 使用指南

### 注册新用户

1. 在登录窗口点击"注册"按钮
2. 填写用户名（至少2字符）和密码（至少4字符）
3. 点击"注册"，成功后可使用新账号登录

### 登录

1. 填写服务器地址（默认localhost）和端口（默认8888）
2. 填写用户名和密码
3. 点击"登录"按钮

### 发送群聊消息

1. 登录后默认处于群聊模式
2. 在底部输入框输入消息
3. 按 Enter 或点击"发送"按钮

### 发送私聊消息

1. 在左侧用户列表点击目标用户
2. 输入框上方会显示"私聊对象: xxx"
3. 输入消息并发送

### 查看聊天记录

- 登录后自动加载最近的群聊历史（最近50条）
- 切换到私聊后自动加载与该用户的私聊历史

## 技术实现

### 数据库表结构

**users 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| username | TEXT | 用户名（唯一） |
| password | TEXT | 密码 |
| is_online | INTEGER | 在线状态 |
| register_time | TIMESTAMP | 注册时间 |
| last_login_time | TIMESTAMP | 最后登录时间 |

**messages 表**
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| sender | TEXT | 发送者 |
| receiver | TEXT | 接收者（群聊为"ALL"） |
| content | TEXT | 消息内容 |
| message_type | TEXT | 消息类型（group/private） |
| timestamp | TIMESTAMP | 发送时间 |

### 消息协议

使用 JSON 格式，每条消息以换行符分隔：

```json
// 登录
{"type": "login", "username": "user1", "password": "1234"}

// 注册
{"type": "register", "username": "user1", "password": "1234"}

// 群聊消息
{"type": "message", "content": "Hello everyone!"}

// 私聊消息
{"type": "private_message", "to": "user2", "content": "Hi!"}

// 群聊消息（服务器广播）
{"type": "group_message", "from": "user1", "to": "ALL", "content": "Hi!", "timestamp": "2024-01-01 12:00:00"}

// 系统消息
{"type": "system_message", "content": "user1 上线了", "timestamp": "2024-01-01 12:00:00"}
```

## 注意事项

1. 服务器默认监听 `0.0.0.0:8888`，可在 server.py 中修改
2. 客户端默认连接 `localhost:8888`
3. 多客户端测试需要在同一局域网或使用端口映射
4. 关闭客户端不会自动关闭服务器
5. 聊天记录保存在 SQLite 数据库文件中

## 常见问题

**Q: 客户端连接失败？**
A: 检查服务器是否启动，地址和端口是否正确。

**Q: 无法注册？**
A: 用户名可能已被占用，请尝试其他用户名。

**Q: 消息发送失败？**
A: 检查是否已登录，网络是否正常。

**Q: 如何退出？**
A: 点击窗口关闭按钮或输入 exit 命令。
